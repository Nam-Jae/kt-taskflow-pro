# 01-product — 제품 정의

## 목표

**"지금 누가 뭐 해?"** 라는 질문이 팀에서 사라지게 한다.

팀 업무를 한 화면에서 시각화해 누구나 현재 진행 상황을 즉시 파악할 수 있도록 한다.
슬랙 DM, 구두 보고, 스프레드시트 없이 업무 흐름이 투명하게 공유된다.

---

## 페르소나

| 항목 | 내용 |
|------|------|
| **역할** | 10명 규모 스타트업 팀리더 |
| **연령대** | 30~40대 |
| **핵심 불편** | 누가 뭘 하는지 파악하려면 매번 물어봐야 함 |
| **기대** | 새 탭 하나를 열었을 때 팀 전체 상황이 한눈에 보임 |
| **환경** | 맥북 + 아이폰, 회의 중 모바일로도 확인 |

---

## MVP 범위

### 핵심 기능

| 기능 | 설명 |
|------|------|
| **Task CRUD** | 추가·목록·수정·삭제 — 4종 모두 화면에서 직접 조작 |
| **상태 분류** | 태스크마다 상태값(예: `todo` / `in_progress` / `done`) 지정 및 표시 |
| **마감 시각 (due_at)** | 날짜 + 시간까지 지정·표시 (예: `2026-05-12 18:00`) |
| **라이트/다크 테마 토글** | 상태는 `localStorage`에 저장, 새로고침 후에도 유지 |
| **모바일 반응형** | 360px 기준 레이아웃 유지 (가장 작은 안드로이드 폰 기준) |

### UI 톤 — macOS 스타일

- 둥근 모서리 (`border-radius`)
- 부드러운 그림자 (`box-shadow`)
- 반투명 카드 (backdrop-filter blur)
- 시스템 폰트 (`-apple-system`, `BlinkMacSystemFont`, `Segoe UI` 폴백)

## MVP 범위 외 (명시적 제외)

| 항목 | 제외 이유 |
|------|-----------|
| 외부 결제 | MVP 수익 모델 없음 |
| 네이티브 앱 (iOS/Android) | 반응형 웹으로 대체 |
| WebRTC (영상·음성) | 복잡도 과다 |
| 외부 캘린더 연동 | 외부 의존성 증가 |
| 파일 업로드 | 스토리지 인프라 필요 |
| WebSocket 실시간 | MVP는 **폴링**으로 대체, 확장 단계에서 도입 |

## 성공 기준 (MVP 완료 조건)

MVP는 아래 기준을 **모두** 통과해야 완료로 간주한다.

| # | 기준 | 측정 방법 |
|---|------|-----------|
| 1 | 새로고침 후 데이터 유지 | 직접 확인 (DB 영속성) |
| 2 | 360px 레이아웃 깨짐 없음 | Chrome DevTools 모바일 시뮬레이터 |
| 3 | API 응답 200ms 이하 | 네트워크 탭 또는 로컬 벤치마크 |
| 4 | CRUD 4종 화면 동작 | 수동 E2E 시나리오 통과 |
| 5 | 라이트/다크 테마 토글 작동 | 토글 후 새로고침 시 상태 유지 확인 |
