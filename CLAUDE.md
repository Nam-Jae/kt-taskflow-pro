## 절대 규칙

다음 5가지는 어떤 이유로도 예외를 두지 않는다.

### 1. 추측 금지
불확실한 사항은 직접 확인하거나 사용자에게 묻는다.
"아마도", "대체로", "보통은" 같은 표현으로 코드를 작성하지 않는다.

### 2. 돌발 의존성 금지
`docs/03_tech-stack.md`에 없는 패키지·라이브러리를 임의로 추가하지 않는다.
새 의존성이 필요하면 먼저 이유와 대안을 설명하고 사용자 승인을 받는다.

### 3. 테스트 없이 완료 금지
구현 코드에는 반드시 대응하는 테스트가 존재해야 완료로 간주한다.
테스트 작성이 불가능한 경우 그 이유를 명시하고 사용자 승인을 받는다.

### 4. 시크릿 하드코딩 금지
API 키, 비밀번호, 토큰, DB 연결 문자열 등을 소스 코드에 직접 작성하지 않는다.
모든 시크릿은 환경 변수(`.env`)로 관리하며, `.env`는 `.gitignore`에 포함한다.

### 5. 폴더 구조 임의 변경 금지
`docs/06_conventions.md`에 정의된 폴더 구조를 임의로 변경하지 않는다.
구조 변경이 필요하면 변경 이유와 영향 범위를 먼저 설명하고 사용자 승인을 받는다.

## 모호한 요청 처리

요청이 불명확하거나 구현 방향이 여러 갈래인 경우, **추측하여 구현하지 않는다.**
대신 선택지 2~3개를 제시하고 사용자가 방향을 결정하도록 한다.

**형식 예시:**
> 다음 중 어떤 방식으로 진행할까요?
> 1. **A 방식** — (장단점 한 줄 요약)
> 2. **B 방식** — (장단점 한 줄 요약)
> 3. **C 방식** — (장단점 한 줄 요약)
